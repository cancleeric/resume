工單管理流程

核心原則

  1. 先開票後修正（發現問題 → 創建工單 → 開始修復）
  2. 單一功能單一工單
  3. 狀態驅動開發
  4. 完整的文檔要求
  5. 工單時數限制：單票不超過 8 小時

狀態流程

open → in_progress → pending_review → reviewed → completed

必填欄位要求

  • modification（修改詳情）: ≥200 字
  • test-results（測試結果）: ≥300 字
  • completion-reason（完成原因）: ≥400 字

評審分數系統（140 分）

項目 分數
Description（需求描述） 20 分
Solution（解決方案） 20 分
Modification（修改詳情） 20 分
Test Results（測試結果） 20 分
Completion Reason（完成原因） 20 分
Test Execution（測試執行） 25 分（選填）
Git Commit（提交記錄） 15 分

通過門檻: ≥126 分（90%）

評審命令

# 新評分系統（推薦）：5×20 + Test Execution 25 + Git 15 = 140
neritic vr submit -r <Ticket ID> -d 20 -s 20 -m 20 -t 20 -c 20 -e 25 -g 15 --result pass --summary "Review comments"

# 舊評分系統（向後相容）：5×25 + Git 15 = 140
